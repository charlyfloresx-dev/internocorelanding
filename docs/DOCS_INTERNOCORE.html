Visión de Arquitectura

InternoCore es un sistema MES (Manufacturing Execution System) diseñado bajo el paradigma de Microservicios Multi-tenant. El objetivo es centralizar la gestión industrial permitiendo que múltiples fábricas operen sobre el mismo núcleo pero con total aislamiento de datos.

Historial de Toma de Decisiones (ADR)

ADR-001: Shared Domain Kernel

Ene 2026

Problema: Duplicidad de lógica de roles y estados en microservicios separados.

Decisión: Creación de la carpeta /common. Todos los microservicios heredan modelos, enums y seguridad de este núcleo.

ADR-002: Motor de Persistencia PostgreSQL

Ene 2026

Problema: Necesidad de indexado eficiente para UUIDs en entornos multi-tenant.

Decisión: Migración de MySQL a PostgreSQL 15 debido a su manejo nativo del tipo de dato UUID y robustez en transacciones concurrentes.

ADR-003: Seguridad Basada en Suscripción

Ene 2026

Problema: Control de accesos dinámico basado en pagos.

Decisión: Middleware universal en security.py que intercepta cada request y valida si la empresa tiene una suscripción active o trial.

Resumen para Documentación de Tesis

  • Implementación de Clean Architecture para separar dominio de infraestructura.
  • Uso de Agnostic UUIDs para garantizar la portabilidad entre motores de base de datos.
  • Automatización de DevOps mediante herramientas de Model Context Protocol (MCP).
  • Seguridad perimetral multi-tenant con validación de Claims de Suscripción en JWT.
