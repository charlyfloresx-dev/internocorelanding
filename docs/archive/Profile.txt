SMK (Universidad - 3 años de expericia) Alto volumen - Baja mezcla.
Se creo el sistema MES (Los principales atributos fueron Conteo de Piezas, tiempo Muerto y Labor) todo el 
ecosistema se desarrollo en PHP, AngularJs y MS SQL, 
se tomo la desicion de utilizar estas tecnologias ya que el servidor que nos proporcionaron
no era potente y por otra parte al hacer modificaciones se reflejaban directamente en produccion.
El primer requerimiento que surgio fue  el conteo de piezas y control de seriales en las cajas,
las piezas tenian un numero de serie y tenian que ser rastreadas en que lote y caja se enviaban.
Al principio solo fue contar piezas y generar etiquetas de las cajas. 
Despues migro junto con el equipo de automatizacion al momento de hacer la pruebas electricas con LavView 
generar un seriado,
consumiendo o enviando informacion a las API de donde y que procesos se le realizo a las piezas.
guardarlo en la base de datos e irlo rastreando en los diferentes procesos y pruebas de calidad el 
cual empezo a generar trazabilidad.
Registrando ademas de tiempos muertos y controles de Labor para validar si el personal contaba con los 
entrenamientos segun el proceso, hubo una conexion entre el sistema TRESS (RH) y 
las validaciones en produccion.
Esto nos permitia generar graficos de informacion para ver en que estaba fallando a la hora de producirse las piezas.
Tambien se creo el sistema de PUll System y celdas flexibles el cual se creo en conjunto con el departamente de Mejora Continua, 
el cual la celda funcionaba deacuerdo al sistema de controles desarrollados, Pruebas electricas, validaciones de procesos previos, 
validacion de numeros seriales, 
delas cuales tenian una configuracion almacenada de los materiales, donde se producian ManPower(Personal) o capacidades necesarias,
se implemento con el almacen un nuevo proceso y procedimientos auxiliado con el sistema, escaners y sensores, 
los cuales indicaban que celdas deberian de surtirse de material conforme avanzaba la produccion, 
para tener listo mucho antes los bines de materiales,
se desarrollo un sistema de control de supermercados, donde en base a lo planeado el almacen deberia tener 
surtidos ciertos bines y alimentado el sistema para que los reconociera.
Para optimizar los servidores se desarrollaron scrips en python que borraban las caches y cerraban conexiones a la base de datos, 
que las pruebas electricas del equipo de automatizacion dejaban colgadas y hacian lenta la red.
Lo cual disminuyo el trafico en la red gracias a las API's, migradas de conexiones directas.
Se implementaron los sistema ANDON por medio de Raspberry PI y servicios web. 
Se desarrollo un sistema de Escalacion y Notificaciones cada que la celda tenia 5min de Tiempo Muerto, 
esto se podia ver en Correo, pantallas y notificaciones SMS al Celular.

ZODIAC AEROSPACE Bajo volumen- Alta mezcla.
La principal funcion era homologar informacion, ya que el ERP que se utiliza es M3 y no es muy amigable su uso,
se opto por descargar reportes almacenarlos en un principio en una computadora y generar graficos, 
para el equipo de altorendimiento, que estaba formado por director de la planta,
gerentes de los diferentes departamentos y yo como especialisate de informacion.
Ya que al ser tres plantas era dificil tener la misma informacion en un solo lugar, ya que se dependia de reportes manuales 
compartidos por correos, los cuales la informacion era incoherente.
Se estandarizo la informacion y se desarrollaron conjunto de graficos que nos mostraban el estado de las plantas (4). 
Gracias a ellos nos dimos cuenta que teniamos mucho material en WIP, 
junto con los materiales cargados a los ordenes de produccion que tenian doble, triple o mas veces cargado el material.
Esto sumaba alrrededor de 4.8 Millones de dolares en materiales perdidos.
Para el cual se desarrollo un sistema de trazabilidad MES (Adaptado completamento al Negocio, casi desde cero con la base de la empreza anterior),
ya que cada order tenia un numero de serie, se podian rastrear los materiales cargados a esa orden,
en que ubicacion de la planta estaba esa pieza y aparte un registro de todas las personas involucradas en sus contruccion por los diferents procesos.
En base al tiempo estandart una vez que la pieza llegaba a embarque, ya se tenia un historial de todo el tiempo invertido en cada uno de los procesos.

Junto con el SWAT Team un equipo multidiciplinarios, gerentes, supervisores, lideres, manufactura, calidad, mejora continua y yo como el desarollador.
En dos meses se logro reducir de 4.8Millones a 40MIl Dlls, muchas de las piezas se lograron encontrar, ya que cada quien en sus proceso debe \
de escanear su orden y su credencial.
y asi se registra todo el tiempo que se le invierte a la pieza. Ademas de mostrarnos cantidad de dinero por procesos y sus areas.
Se crearon controles para surtidos de materiales, regitros de pruebas de calidad, 
donde cada estacion tenia registrada la informacion de las ordenes procesadas y pendientes a procesar,
informacion detallada sobre los kit de materiales que se cargarian a las ordenes, 
Informacion sobre las personas trabajando, tiempo disponible del turno y tiempos de descanzo, metas, tiempo muerto, paros programados, 
todo esto nos daba el tiempo real de operacion y lo planeado para cumplir con el plan de produccion.
Ademas de generar la informacion de OOE, La Eficiencia Total, Tiempo Productivo, FirstPastYield(Calidad), TakTimes.

Para el cual se crearon varias herramientas desarrolladas en un entorno web accesibles desde cualquier dispositivo con acceso a la red.
Como ya tenia la experiencia previa, todo se desarrollo en API .NET CORE el cual trabaja directo en el Kernel para hacer mas eficiente 
aun el procesamiento,
Las API's eran consumidas por las diferentes aplicaciones, al final todo se generalizaba para la generacion de nuevos reportes, 
pantallas de informacion y notificaciones.
Igual en correo electronicos, Mensajes SMS y pantallas en las areas de produccion. 
Una vez estable el sistema me movieron al equipo de manufactura, en el cual estube generando nuevos procesos auxiliado 
por los sistemas que desarrollaba, 
como controles de inventarios, registros de calidad, medicion de PLC's (algunas maquinas no todas se tenia el acceso).
Se crearon nuevas hojas de trabajo y el procesos de produccion se adapto a los sistemas y controles desarrollados. 
COntroles de FreeStock (Materiales), toolcrips (Herramientas necesarias para la produccion)
Las herramientas utilizadas fueron API .NET CORE, AngularJS para la interfaz del usuarios, y MYSQl en la Base de datos. 
Ya que al inicio no se contaba con el recurso necesario.
Tambien se estubo desarrollando un anexo al sistema de trazabilidad mediante RFID's aunque ya se podia tener un mejor control de las piezas 
y materiales aun teniamos que localizar piezas,
este proyecto no se completo debilido a la falta de presupuesto, se tenia que invertir en antenas RFID etiquetas y 
aparte no cumplia con especificaciones aeronauticas, ya que al integrar
frecuencias a las piezas estas podian interferir con los intrumentos de aviacion.
Al Final la pieza no se podia escanear en un proceso si su hoja de ruta no era valida y el proceso anterior no tenia informacion registrada.
Se logro obtener registros de sobre retrabajos asi como, multiples cargas de materiales que se le agregaban a las ordenes.
Ahi era donde se estaba perdiendo todo el dinero en materiales y desperdiciando tiempo de produccion, se crearon controles de scrap, 
el cual mostraba a los equipo cuales eran principales issues de calidad.
Al tener todos esta informacion graficada y clasificada para su facil entendiemiento se mejoraban los procesos de produccion.
